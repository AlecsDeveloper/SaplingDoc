Introduccion


¿Qué es?


Sapling es un complemento para Minecraft Bedrock Edition el cual proporciona comandos internos dentro del juego, cada uno con sub-comandos con argumentos.

Siendo este complemento enfocado a la información de los mundos, generación de jugadores y agregado de nuevas funciones al juego.



¿Cómo funciona?


Sapling está escrito completamente en JavaScript dentro de una API de Scripting del propio juego. Sapling se conforma por un núcleo principal que es capaz de proporcionar algunos de los siguientes aspectos:

- Lectura NBT
- Empaquetador de paquetes de comandos avanzados
- Envío de mensajes desde el servidor al cliente.

Sapling corre en una instancia Gametest la cual permite hacer la incorporación de la lectura de datos y generación de jugadores, además de proporcionar una capa extra de modificación del juego, puesto que dicha instancia cuenta con mayores funciones a nivel interno dentro de la Scripting API.



Instancia Gametest


Crear una instancia Gametest dentro de la Scripting API es bastante sencillo, a continuación habrá un ejemplo del codigo inicial.



 import * as
GameTest from
'@minecraft/server-gametest'

 GameTest.Register('Namespace', 'Name', event => {
    /* Codigo dentro de la instancia */
 })
    .maxTicks(999999999)
    .structureName('SP:Instancia');


Como se puede observar, la instancia regresa un callback definido como event el cual regresará 3 métodos especiales los cuales son:

  - SimulatedPlayer
  - FenceConnectivity
  - SculkSpreader
